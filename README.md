> **Кратко:** Мета-система обучения для погружения в разработку и интеграцию ИИ-агентов. От основ Python до архитектуры роев агентов.

---

## 📖 О проекте

Это не просто промпт, а **структурированный обучающий комплекс**. В мире, где технологии меняются ежемесячно, сложно найти актуальный путь. Этот проект решает проблему хаотичного обучения, выступая в роли **персонального методиста**, который координирует работу узкопрофильных «учителей».

### 💡 Основная идея
Создать экспериментальный инструмент, который помогает не только писать код, но и понимать:
*   Архитектуру ИИ-систем.
*   Бизнес-интеграцию.
*   Сложные взаимодействия в роях агентов.

---

## ⚡ Почему это работает?

1.  **Модульность для экономии контекста**  
    Вместо одного гигантского промпта система генерирует специализированные промпты для каждого этапа. Вы работаете с «Учителем» в отдельном чате, а «Методист» хранит общую карту прогресса.
2.  **Фокус на методах и инструментах (Новинка v1.2)**  
    Перед каждым практическим заданием учитель обязан выдать список ключевых методов/функций. Больше никакой магии «я не знаю, чем это решать».
3.  **Педагогические принципы**  
    *   🧠 **Техника Фейнмана:** Объяснения простым языком с бытовыми аналогиями.
    *   🔢 **Правило трёх единиц:** Не больше 3 новых фактов за блок.
    *   🛠 **Интерактивность:** Тесты, задачи, код пишете вы (учитель только направляет).
4.  **Честный трекинг**  
    Система отчётов позволяет фиксировать прогресс. Это создаёт «карту знаний», к которой можно вернуться для повторения.

## ⚠️ Важные особенности использования

> [!WARNING]
> **Ручной перенос отчётов**  
> Поскольку это промпт, а не полноценное приложение, система не имеет памяти между сессиями автоматически.
> 1. После завершения темы **Учитель генерирует Отчёт**.
> 2. Вы **копируете его** и отправляете **Методисту**.
> 3. Это обновляет «Карту прогресса». Без отчёта Методист не знает, что вы прошли тему.

> [!TIP]
> **Разделение чатов**  
> Для экономии контекста рекомендуется запускать промпт Учителя (который выдаст Методист) в **новой вкладке/чате**. В основном чате остаётся только Методист.

## 📖 About the Project

This is not just a prompt — it's a **structured learning complex** designed to solve the problem of chaotic self-education in the fast-changing world of AI. 

Acting as your **personal methodologist**, it coordinates specialized "teachers" (Python, API, RAG, Agents) to guide you from beginner to practitioner in AI agent development.

### 💡 Core Idea
Create an experimental learning tool that helps you not only write code, but also understand:
*   🏗️ AI system architecture
*   🔗 Business process integration
*   🐝 Complex interactions in agent swarms

---

## ⚡ Why It Works

| Feature | Benefit |
|---------|---------|
| **🧩 Modular Design** | Instead of one massive prompt, the system generates specialized prompts per stage. Work with a "Teacher" in a separate chat while the "Methodologist" tracks your overall progress. |
| **🛠 Method-First Approach (v1.2)** | Before each practical task, the Teacher *must* provide a list of key methods/functions. No more guessing "what tools should I use?". |
| **🧠 Pedagogical Principles** | • **Feynman Technique**: Simple explanations with real-life analogies<br>• **Rule of Three**: Max 3 new concepts per block<br>• **Interactive Learning**: Quizzes, tasks, *you* write the code |
| **🗺 Honest Progress Tracking** | Manual report system creates a persistent "knowledge map" you can revisit for review without reloading context. |

---

## 🏗 System Architecture

The system consists of a **Main Prompt (Methodologist)** and dynamically generated **Teacher Prompts**.

| Role | Description |

| 🐍 **Python Teacher** | Foundation: variables → async programming. |

## ⚠️ Important Usage Notes

> [!WARNING]
> ### Manual Report Transfer Required
> Since this is a prompt-based system (not a standalone app), progress is **not auto-saved** between sessions.
> 1. After completing a topic, the **Teacher generates a Report**.
> 2. **You copy-paste** this report to the **Methodologist**.
> 3. This updates your **Progress Map**. Without the report, the Methodologist cannot track your completion.

> [!TIP]
> ### Use Separate Chats for Efficiency
> To conserve context window:
> - Run the **Teacher prompt** (provided by Methodologist) in a **new chat/tab**.
> - Keep the **Methodologist** in your main chat for coordination.
> This prevents context overflow and keeps sessions focused.
