📖 О проекте
Это не просто промпт, а мета-система обучения, разработанная для погружения в разработку и интеграцию ИИ-агентов с нуля до уровня практика.
В мире, где технологии меняются ежемесячно, сложно найти структурированный путь. Этот комплекс решает проблему хаотичного обучения, выступая в роли персонального методиста, который координирует работу узкопрофильных «учителей» (Python, API, RAG, Agents).
Идея проекта: Создать экспериментальный обучающий инструмент, который помогает не только писать код, но и понимать архитектуру, бизнес-интеграцию и сложные взаимодействия в роях агентов.
⚡ Почему это работает?
Модульность для экономии контекста.
Вместо одного гигантского промпта, который не влезет в окно контекста, система генерирует специализированные промпты для каждого этапа. Вы работаете с «Учителем» в отдельном чате, а «Методист» хранит общую карту прогресса.
Фокус на методах и инструментах.
Новинка v1.2: Перед каждым практическим заданием учитель обязан выдать список ключевых методов/функций. Больше никакой магии «я не знаю, чем это решать» — вы заранее видите инструментарий.
Педагогические принципы.
Техника Фейнмана: Объяснения простым языком с бытовыми аналогиями.
Правило трёх единиц: Не больше 3 новых фактов за блок.
Интерактивность: Тесты, задачи, код пишете вы (учитель только направляет).
Честный трекинг.
Система отчётов позволяет вручную (copy-paste) фиксировать прогресс. Это создаёт «карту знаний», к которой можно вернуться для повторения без загрузки нового контекста.
🏗 Архитектура комплекса
Система состоит из Главного Промпта (Методист) и генерируемых им Промптов Учителей.
mermaid

Роли в системе
Методист (Коуч): Центральный узел. Не учит, а управляет. Анализирует отчёты, ведёт карту, рекомендует следующий шаг.
Учитель Python: Фундамент (от переменных до асинхронности).
Учитель API: Работа с HTTP, LLM API, российскими сервисами (GigaChat, YandexGPT).
Учитель RAG: Векторные базы, поиск, оценка качества.
Учитель Agents: LangChain, LangGraph, память, планирование, интеграция в бизнес.
Исследователь Роев: Продвинутый блок. Эксперименты, обзор новых репозиториев, архитектура мультиагентных систем.

📚 Программа обучения (Roadmap)
Шаг 1. Python (0 → Инженер)
Этап 1: Фундамент (типы, циклы, функции). Проект: Консольный ежедневник.
Этап 2: Инструментарий (Git, JSON, OOP). Проект: Анализатор логов.
Этап 3: Продвинутый (Async, API, Decorators). Проект: Telegram-бот.
Этап 4: AI-Ready (Docker, FastAPI). Проект: Микросервис анализа текста.
Шаг 2. API & Интеграции
HTTP/REST, Auth (OAuth2, Keys).
Работа с LLM API (OpenAI, Anthropic, RF Models).
Function Calling, Streaming, Async requests.
Создание своего API на FastAPI.
Шаг 3. RAG (Retrieval-Augmented Generation)
Векторные БД (Chroma, Qdrant).
Чанкинг, гибридный поиск, реранкинг.
Оценка качества (RAGAS).
Шаг 4. Разработка ИИ-Агентов
Фреймворки (LangChain, LangGraph, CrewAI).
Память, Планирование (ReAct), Инструменты.
Внедрение в бизнес-процессы, мониторинг.
Шаг 5. Исследование Роев (Experimental)
Обзоры новых фреймворков и статей (arXiv).
Разбор чужого кода сложных систем.
Эксперименты с архитектурой роя (синхронизация, конфликты).
⚠️ Важные особенности использования
Ручной перенос отчётов.
Поскольку это промпт, а не полноценное приложение, система не имеет памяти между сессиями автоматически.
Как работает: После завершения темы Учитель генерирует Отчёт. Вы копируете его и отправляете Методисту.
Зачем: Это обновляет «Карту прогресса» Методиста. Без отчёта Методист не знает, что вы прошли тему.
Разделение чатов.
Для экономии контекста рекомендуется запускать промпт Учителя (который выдаст Методист) в новой вкладке/чате. В основном чате остаётся только Методист.
Обратная связь.
Если Учитель забыл выдать отчёт — напомните ему. Это критично для работы системы.
